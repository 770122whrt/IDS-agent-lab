Bridge IDS Reference Requirements - Manual Verification Test
================================================================

This file contains 15 manually written natural language requirements based on the
20250317023029_IFCBridge-IDS-xml.ids file. These requirements will be used to test
the IDS generation pipeline and compare outputs against the reference IDS.

Domain: Bridge Infrastructure (IFC4X3_ADD2)
Total Reference Specifications: 45
Selected for Testing: 15 (covering different complexity levels)

================================================================
REQUIREMENTS
================================================================

1. All bridge projects must contain exactly one IfcProject instance that identifies the overall project and provides defaults for units.

2. Each bridge dataset should include at least one IfcSite that defines the terrain within vicinity of the bridge project. The site must be aggregated under the IfcProject and must have a Name attribute.

3. Every bridge structure must be represented by an IfcBridge entity with a required Name attribute. The bridge must be aggregated under either another IfcBridge or the IfcProject.

4. Horizontal and vertical alignment curves must be defined using IfcAlignment entities. Each alignment must be aggregated under the IfcProject and must have a Name attribute.

5. The substructure of the bridge, which transfers loads into the ground, must be modeled as IfcBridgePart with predefined type SUBSTRUCTURE. It must be aggregated under the IfcBridge.

6. The superstructure of the bridge, which spans horizontally to carry loads onto substructures, must be modeled as IfcBridgePart with predefined type SUPERSTRUCTURE. It must be aggregated under the IfcBridge and must have a Name attribute.

7. Bridge girders should be modeled as IfcElementAssembly with predefined type GIRDER. They must be contained within the SUPERSTRUCTURE bridge part, have a Name attribute, and must specify material information.

8. The bridge deck, which conveys traffic but does not perform structural functions, must be modeled as IfcBridgePart with predefined type DECK. It must be aggregated under the IfcBridge and must have a Name attribute.

9. Deck segments should be modeled as IfcBridgePart with predefined type DECK_SEGMENT. They must be aggregated under the DECK bridge part and must have a Name attribute.

10. Abutments at the ends of the bridge must be modeled as IfcBridgePart with predefined type ABUTMENT. They must be aggregated under the SUBSTRUCTURE bridge part and must have a Name attribute.

11. Bridge piers must be modeled as IfcBridgePart with predefined type PIER. They must be aggregated under the SUBSTRUCTURE bridge part and must have a Name attribute.

12. Foundation piles must be modeled as IfcPile entities. They must be contained within the SUBSTRUCTURE bridge part, have a Name attribute, and must specify material information.

13. The proposed ground surface indicating final site conditions must be modeled as IfcGeographicElement with predefined type TERRAIN. It must be contained within the IfcSite and must have a Name attribute.

14. The existing ground surface indicating current site conditions must be modeled as IfcGeographicElement with predefined type TERRAIN. It must be contained within the IfcSite and must have a Name attribute.

15. Boring points indicating soil layers and properties at discrete locations must be modeled as IfcGeographicElement. They must be contained within the IfcSite, have a Name attribute, and must include the property "Depth" in the property set "Pset_GeographicElementCommon".

================================================================
NOTES
================================================================

Coverage:
- Spatial structure: Project, Site, Bridge (specs 1-3)
- Alignment: Horizontal/vertical curves (spec 4)
- Bridge parts: Substructure, Superstructure, Deck (specs 5-6, 8)
- Structural elements: Girders, Piles (specs 7, 12)
- Detailed components: Deck segments, Abutments, Piers (specs 9-11)
- Site elements: Proposed/Existing terrain, Boring points (specs 13-15)

Complexity levels:
- Simple (entity only): specs 1, 5
- Medium (entity + attribute): specs 2, 3, 4, 6, 8-11
- Complex (entity + attribute + material): specs 7, 12
- Advanced (entity + attribute + property): spec 15

Expected mapping challenges:
- Spec 7: Girders → IfcElementAssembly (not IfcBeam)
- Spec 9: Deck segments → IfcBridgePart (not IfcDuctSegment)
- Spec 12: Piles → IfcPile (not IfcColumn)
- Spec 13-14: Terrain → IfcGeographicElement with predefinedType
- Spec 15: Property requirements on IfcGeographicElement
