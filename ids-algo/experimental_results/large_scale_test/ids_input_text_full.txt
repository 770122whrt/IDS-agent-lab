【Specification 1】Project
IFC Version: IFC4X3_ADD2
Description: All data sets shall consist of a single IfcProject instance, which identifies the overall project, provides defaults for units, and holds a graph of references to all data within scope.

Can't see any possibilities to have unit requirements in IDS.
The RAR spoecifies:
- Length Unit: Default unit for length. For imperial, inches are recommended. SI Meters are thedefault if not provided.
- Angle Unit: Default unit for angles. Degrees are recommended, and are the default if not provided.
- Mass Unit: Default unit for mass. For imperial, pounds are recommended. SI Kilograms are the default if not provided.
Temperature Unit: Default unit for temperature. For imperial, fahrenheit is recommended. SI Celcius is the default if not provided.

Applicability:
  Cardinality: 1..unbounded
  • Applies to all IFCPROJECT type entities

Requirements:

【Specification 2】Site
IFC Version: IFC4X3_ADD2
Description: Data sets shall include at least one site. The site may define existing and proposed terrain within vicinity of the bridge project and contain alignment(s), bridge(s) and potentially other structures such as the road/rail supported by the bridge. Boring points may be included as objects contained within the site.

Comment: The RAR specifies a mandatory Site. However here it is made optional. Also the reference elevation is required in the RAR. However, the reference elevation should be present in the CRS reference.

Applicability:
  Cardinality: 0..unbounded
  • Applies to all IFCSITE type entities

Requirements:
  1. Part Of Requirement
     - Relation Type: IFCRELAGGREGATES
     - Related Entity: IFCPROJECT
     - Cardinality: Required
  2. Attribute Requirement
     - Attribute Name: Name
     - Cardinality: Required
     - Instructions: Name of the site for referencing purposes.

【Specification 11】Piers
IFC Version: IFC4X3_ADD2
Description: Piers are decomposed into elements according to connectivity, indicating construction joints. Reinforcing may be indicated within inner elements such as footings, columns, members, and walls (see documentation at corresponding elements for usage); such reinforcing shouldreflect how it is to be placed at time of construction such that rebar connecting between elements is projected out of the element where it is initially placed.
Abutments and piers are placed relative to the horizontal alignment curve (NOT the vertical alignment curve), with components placed according to Cartesian placement within. This reflects positioning as typically indicated on construction plans, where all dimension lines are based on Cartesian positioning relative to the position and orientation of the station along the horizontal alignment curve.

Missing: Alignment, Location (not possible with IDS)

【Specification 16】Material
IFC Version: IFC4X3_ADD2
Description: Materials are defined on elements to be constructed (e.g. concrete), fabricated (e.g. steel girders), and that exist on site (e.g. soil borings), indicating material category, classification, and structural properties.

Applicability:
  Cardinality: 1..unbounded
  • Applies to all IFCMATERIAL type entities

Requirements:
  1. Attribute Requirement
     - Attribute Name: Name
     - Cardinality: Required
     - Instructions: Material classification according to the respective authority (e.g. ASTM)
  2. Attribute Requirement
     - Attribute Name: Category
     - Value: Must be one of: Concrete, Steel, Wood, Plastic, Glass, Earth
     - Cardinality: Required
     - Instructions: Category of material, where if provided must be one of "Steel", "Concrete", "Wood", "Plastic", "Glass", "Earth".
  3. Property Requirement
     - Property Set: Pset_MaterialCommon
     - Property Name: MassDensity
     - Data Type: IFCMASSDENSITYMEASURE
     - Cardinality: Optional
     - Instructions: Material mass density.
  4. Property Requirement
     - Property Set: Pset_MaterialMechanical
     - Property Name: YoungModulus
     - Data Type: IFCMODULUSOFELASTICITYMEASURE
     - Cardinality: Optional
     - Instructions: A measure of the Young's modulus of elasticity of the material.
  5. Property Requirement
     - Property Set: Pset_MaterialMechanical
     - Property Name: ShearModulus
     - Data Type: IFCMODULUSOFELASTICITYMEASURE
     - Cardinality: Optional
     - Instructions: A measure of the shear modulus of elasticity of the material.
  6. Property Requirement
     - Property Set: Pset_MaterialMechanical
     - Property Name: ThermalExpansionCoefficient
     - Data Type: IFCTHERMALEXPANSIONCOEFFICIENTMEASURE
     - Cardinality: Optional
     - Instructions: A measure of the expansion coefficient for warming up the material about one Kelvin.
  7. Property Requirement
     - Property Set: Pset_MaterialConcrete
     - Property Name: CompressiveStrength
     - Data Type: IFCPRESSUREMEASURE
     - Cardinality: Optional
     - Instructions: [If Category=”Concrete”] The compressive strength of the concrete.
  8. Property Requirement
     - Property Set: Pset_MaterialSteel
     - Property Name: YieldStress
     - Data Type: IFCPRESSUREMEASURE
     - Cardinality: Optional
     - Instructions: [If Category=”Steel”] A measure of the yield stress (or characteristic 0.2 percent proof stress) of the material.
