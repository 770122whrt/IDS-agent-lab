## 一、可用Skills工具箱

### 🎯 核心Skills（必用）

#### 0. **docx** ⭐⭐⭐⭐ 最关键
**功能**：读取/编辑Word文档，支持追踪修订模式
- 读取.docx文件内容
- 以追踪修订模式（Tracked Changes）标记所有修改
- 所有修改以"Claude"作为作者标识
- 支持插入、删除、替换等操作

**使用场景**：
- 读取`chapter_01_draft.docx`原始内容
- 将所有修改以审阅模式写回Word文档

**调用方式**：
```bash
/docx [具体任务描述]
```

**技术细节**：
- 解包Word文档为XML
- 编辑XML添加追踪修订标记
- 重新打包为.docx文件

---

#### 1. **humanizer** ⭐⭐⭐ 最重要
**功能**：去除AI写作痕迹
- 识别并修复7类AI写作模式
- 降低AI高频词汇密度
- 打破三段式结构
- 增加人性化表达

**使用场景**：
- 第一步：全章去AI化处理
- 最后一步：最终验证

**调用方式**：
```bash
/humanizer [输入第一章全文]
```

---

#### 2. **research-writing-prompts** ⭐⭐⭐ 最实用
**功能**：学术写作提示词模板（来自MSRA、Seed、上海AI Lab）

**可用功能列表**：
| 功能 | 触发词 | 用途 |
|------|--------|------|
| 中文润色 | "润色中文" | 优化中文学术表达 |
| 逻辑检查 | "检查逻辑" | 检查论证逻辑 |
| 去AI味(中文) | "去AI味中文" | 去除中文AI痕迹 |
| 缩写 | "缩写" | 压缩冗长表述 |
| 扩写 | "扩写" | 扩充内容 |

**使用场景**：
- 第二步：中文润色
- 第三步：逻辑检查
- 第四步：再次去AI味

**调用方式**：
```bash
/research-writing-prompts [具体任务] [输入文本]
```

---

#### 3. **ml-paper-writing** ⭐⭐ 重要
**功能**：ML/AI论文写作指导
- 引用验证（防止幻觉引用）
- 论文结构指导
- 学术规范检查

**使用场景**：
- 补充文献引用时验证引用准确性
- 检查学术规范

**调用方式**：
```bash

```

## 三、执行顺序建议

### 方案A：一次性完成（推荐）⭐
**适合场景**：有5-6小时连续时间

**执行顺序**：
```
Task 1.1 → Task 1.2 → Task 2.1 → Task 2.2 → Task 2.3 → Task 2.4 
→ Task 3.1 → Task 3.2 → Task 4.1 → Task 4.2 → Task 4.3 → Task 5.1 → Task 5.2
```

**优点**：
- 一气呵成，逻辑连贯
- 术语统一更容易
- 总时间最短

---

### 方案B：分两次完成
**适合场景**：只有碎片化时间

**第一次（3小时）**：
```
Task 1.1 → Task 1.2 → Task 2.1 → Task 2.2 → Task 2.3 → Task 2.4
```

**第二次（2.5小时）**：
```
Task 3.1 → Task 3.2 → Task 4.1 → Task 4.2 → Task 4.3 → Task 5.1 → Task 5.2
```

---

### 方案C：分三次完成
**适合场景**：每次只有1-2小时

**第一次（2小时）**：去AI化
```
Task 1.1 → Task 1.2
```

**第二次（1.5小时）**：内容增强
```
Task 2.1 → Task 2.2 → Task 2.3 → Task 2.4
```

**第三次（2小时）**：润色与验证
```
Task 3.1 → Task 3.2 → Task 4.1 → Task 4.2 → Task 4.3 → Task 5.1 → Task 5.2
```
